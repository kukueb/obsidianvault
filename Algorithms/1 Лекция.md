# Информационные структуры
Пусть $D = \{d_1, d_2, ..., d_n\}$ - некоторое множество переменных (в общем случае бесконечное) и пусть $R = \{r_1, r_2, ..., r_m\}$ - множество отношений, заданных на множестве $D$, тогда $I$ = <$D,R$> - называется **информационной структурой** или **структурой данных**
Элементы множества $D$ называются **узлами информационной структуры**. Они могут иметь любой тип, в том числе и структурированный.

Отдельным классом информационных структур являются линейные информационные структуры, на узлах которых задано отношение **линейного порядка**.
У таких структур определён *первый и последний элемент*. А также определена *операция доступа к элементу* (примером является массив).

Различают **статические** и **динамические** структуры. В *статических* информационных структурах количество и тип элементов - **неизменны**.

Для динамической информационной структуры кроме операции инициализации *(создания структуры)* определены операции **добавления** и **удаления** узлов.
Следует различать **логическую организацию** *(как работает структура, правила работы)* и **физическую реализацию** информационных структур.

## Динамические линейные информационные структуры
### Стек 
это линейная информационная структура, доступ к элементам которой определяется по принципу **последним пришёл, первым вышел** *(Last Input First Output)*

| верхушка |
| -------- |
|          |
|          |
|          |
| **дно**  |
**Операции над стеком:**
1) *Добавить узел*
2) *Удалить узел*
3) *Создать стек*
4) *Проверить стек на пустоту*
Существует понятие **Абстрактного стека**, размер которого *неограничен*. Однако на практике работают только с **Ограниченным стеком** и поэтому можно добавлять операцию - *Стек полон* (проверки на заполненность)
### Очередь
это линейная информационная структура, доступ к элементам которой выполняется по принципу **первый пришёл, первый вышел** *(First Input First Output)*.
В очереди существует указатель начала очереди и конца очереди.


|     |     |     |     |     |     |
| --- | --- | --- | --- | --- | --- |
добавить                                                  удалить

**Операции:**
1) *Инициализация*
2) *Очередь пуста*
3) *Добавление элемента в конец*
4) *Удаление узла из начала очереди*
Так же существует понятие **абстрактной** и **ограниченной** очереди
**Синхронная очередь** (очередь за булочкой)
**Асинхронная очередь** ("мне только спросить" в поликлинике)
### Дек (Deque)
это линейная информационная структура, доступ к элементам которого происходит с **обоих концов**.
**Операции:**
1) *Инициализация*
2) *Проверка дека на пустоту*
3) *Добавление узла в начало*
4) *Добавление узла в конец*
5) *Удаление узла из начала*
6) *Удаление узла из конца*

### Реализация
#### Моделирование стека (через массив)
```c++
#include <iostream>
#define max 20

struct Stek {
  int top;
  Node s[max];
};

Stek St;
void InitStek(Stek &St) {
  St.top = -1;
}

void Push(Stek &St; Node x) {
  if (St.top == max) cout << "переполнение стека";
  else { St.top++; St.s[St.top] = x; }
  // else St.s[++St.top] = x;
}

bool Empty(Stek St) {
  if (St.top == -1) return 1;
  else return 0;
}

void Pop(Stek &St, Node &x) {
  if (Empty(St) == 1) cout << "Стек пуст";
  else { x = St.s[St.top]; --St.top; }
}
```
К стандартному набору операций стека часто добавляют **Метод top** - обращение к элементу на верхушке стека
```c++
Node top(Stek &St) {
	int i = St.top;
	return St.s[i];
}
```
**Метод size()**  - позволяет определить количество элементов в стеке
```c++
int size(Stek &St) {
	return (st.top);
}
```

### Грамматика правильной скобочной последовательности
Правильная скобочная последовательность — последовательность, состоящая из символов - «скобок», в которой каждой отрывающейся скобке соответствует закрывающаяся скобка такого же типа, что и открывающаяся скобка.
Например, правильными будут следующие последовательности: \[(\[])(((\[\[]])))){()}. Не будут являться правильными скобочные последовательности \[\[)) (несоответствие типа закрывающихся скобок типу открывающихся), } (закрывающая скобка стоит раньше открывающейся), \[\[{()] (не каждой открывающейся скобке соответствует закрывающаяся).
```c++
#include <string>
using namespace std;
int main() {
  string s;
  Stek b;
  getline(cin, s);

for (auto c = s.begin(); c != s.end(); c++) {
  if (*c==')' && b.top() == '(' || *c==']' && b.top() == '[' || *c=='}' && b.top() == '{')
    b.pop();
  else if (*c==')' || *c==']' ||*c=='}') {
    cout << "NO" << endl;
    return 0;
  }
  else b.push(*c);
}
b.empty()?cout << "YES": cout << "NO";
return 0
}
```
