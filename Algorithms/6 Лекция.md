## Обходы деревьев
Под обходом дерева понимается последовательное посещение узлов с некоторой обработкой их информационной составляющей

Существуют 3 основных обхода:
1) **Прямой или П-обход**. Состоит в посещении корня, П-обхода левого поддерева и П-обхода правого поддерева. (Корень, левый, правый - КЛП)
2) **Внутренний обход или В-обход**. Состоит в В-обходе левого поддерева, корня и В-обхода правого поддерева (Левый, корень, правый - ЛКП) - *Один из лучших способов сортировки чисел, без учёта эффективности устройства структуры данных*
3) **Обратный обход или О-обход**. Состоит в посещении левого поддерева, затем в посещении правого поддерева и корня. (ЛПК)

```c++
void print(tree* p) { // КЛП
	if (p!=NULL) {
		cout << "\nElem: " << p->info; // корень
		cout << "\n----------------\n";
		print(p->L); // Левый
		print(p->R); // Правый
	}
}
```

### Удаление узла из двоичного дерева поиска
При удалении существует 3 ситуации:
1) У удаляемой вершины нет сыновей (хотим удалить лист)
2) У вершины 1 потомок. Тогда выполняется замена на этого потомка
3) У вершины 2 потомка. Необходимо заменить вершину на максимум левого поддерева, а затем удалить этот максимум.

```c++
tree *delete_node(tree *t, int data) {
  if (t == nullptr)
    return t;
  if (data == t->info) { // удаляемый узел
    tree *tmp;
    if (t->R == nullptr)
      tmp = t->L;
    else {
      tree *ptr = t->R;
      if (ptr->L == nullptr) {
        ptr->L = t->L;
        tmp = ptr;
      } else {
        tree *pmin = ptr->L;
        while (pmin->L != nullptr) {
          ptr = pmin;
          pmin = ptr->L;
        }
        ptr->L = pmin->R;
        pmin->L = t->L;
        pmin->R = t->R;
        tmp = pmin;
      }
    }
    delete t;
    return tmp;
  }
  else if (data < t->info) 
  t->L = delete_node(t->L, data);
  else t->R = delete_node(t->R, data);
  return t;
}
```

# Ввод-вывод 
в большинстве языков программирования ввод-вывод связан с понятием файла. Файл - это поименованная совокупность данных, объединённых общим свойством.

Необходимость использования файлов связана с длительным хранением данных и большим объёмом
## Система управления вводом-выводом
Для организации ввода-вывода, необходимо выполнить некоторую последовательность действий, которые удобно записать 1 раз в виде подпрограммы и вызывать многократно.

Набор таких подпрограмм называется **системой управления вводом-выводом (СУВВ)**

СУВВ состоит из **логической** и **физической** части.

Средства физического уровня позволяют работать с устройствами ввода-вывода и делать эту работу эффективной.

Логический уровень СУВВ связан с обработкой файлов и предоставляет программисту средства управления, например, буфером.

**BIOS - СУВВ!!!**
