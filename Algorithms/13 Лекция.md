# Введение в компиляторы
Компилятор состоит из блока **Анализа** и **Синтеза**.

На этапе анализа проверяется синтаксическая и семантическая корректность программы, выявляется её структура и формируется промежуточная форма представления программы.

На этапе синтеза выполняются подготовительные действия по оптимизации промежуточной формы и генерация объектного кода.

На вход компилятора поступает исходная программа (текстовый файл).
На первом этапе её обрабатывает **сканер**. Он преобразует отдельные символы в **лексемы**, используя разделители

Разделители находятся в **информационных таблицах**.
Кроме того в информационные таблицы либо помещаются лексемы, либо они сравниваются с ключевыми словами.

## Структура информационных таблиц
Любая таблица состоит из двух полей: *аргумент* и *значение*

| Arg | Val |
| --- | --- |
Поле **аргумент** чаще всего содержит **идентификатор** (ключевой слово, разделитель, имя переменной)
Поле **значение** содержит характеристики 
```
int a;
iff (a > 0) a = [2][3];
```

| разделители | идентификаторы<br>arg | идентификаторы<br>val (кол-во байт) | лексемы | константы |
| ----------- | --------------------- | ----------------------------------- | ------- | --------- |
| ` `         | a                     | 4                                   | for     | =         |
| ;           | iff                   |                                     | if      | ==        |
| (           |                       |                                     | int     | >         |
|             |                       |                                     |         | <         |
|             |                       |                                     |         | 0         |

Синтаксический анализатор за счёт информационных таблиц выявляет корректность описания объектов программы, укрупняет лексические единицы в синтаксические конструкции (операторы) и проверяет их корректность

## Семантический анализатор
Для поиска семантических ошибок разрабатываются семантические подпрограммы, которые описывают закономерность работы распознанных синтаксических единиц.

## Промежуточная форма представления программ
Существует 2 способа:
1) Дерево вывода
2) Табличная форма (**триады** или **тетрады**)
	`A := A+B*2,5`
	
	| Операция | 1 операнд | 2 операнд | Результат |
	| -------- | --------- | --------- | --------- |
	| *        | B         | 2,5       | R1        |
	| +        | A         | R1        | R2        |
	| :=       | A         | R2        | -         |

### Хеш-функции
Пусть на множестве идентификаторов некоторого ЯП задана функция $f$, принимающая значения из некоторого диапазона $1 ..N$, такая что для двух различных идентификаторов, значение различно (т.е. $f$ - обратима)

Такую функцию называют функцией прямого доступа.

Использование такой функции при организации таблиц приведёт к неограниченному размеру.
На практике условие 1 может нарушаться (разные лексемы - разные значения), такие функции называются **Хеш-функциями** и при разрешении коллизий используется один из методов
#### 1 подход разрешения коллизии - Рехеширования
Для разрешения коллизий можно использовать рехеширование
Если для некоторого идентификатора $S$ позиция $f(S)$ - уже занята, то выбирается позиция $f(S)+R_{1}$. Причём необходимо использовать формулу $(f(S)+R_{i})\; \mathrm{mod}\; N$ (для того, чтобы не выйти за пределы)
1) линейное рехеширование; здесь $R_{i} = I$, т.е. $(f(S)+I)\;\mathrm{mod}\;N$
2) рехеширование сложением; $R_{i} = i*f(S)$, т.е. $(f(S)+i\cdot f(S))\;\mathrm{mod}\;N$
3) случайное рехеширование. На каждом шаге рехеширование $R_{i}$ - некоторая случайная

#### 2 подход - Метод цепочек переполнения
Для каждой строки таблицы идентификаторов добавляется поле `Link`, представляющее собой связный список, обеспечивающий связь между лексемами попавшими в коллизию, но к таблице необходимо добавить массив начальных адресов этих цепочек.
