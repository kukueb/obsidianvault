# Блок FCB
Для каждого файла операционная система выделяет специальную область памяти, размером 256 байт, называемую FCB (File Control Block)
	В этот блок заносится **номер диска**, **имя** и **тип файла** (идентификатор файла), **размер файла** и **дата** (атрибуты файла), **длина записи файла**, которая используется при операциях чтения/записи.

# Буферизация ввода-вывода
Буферизация - это приём программирования, состоящий в опережающей подготовке данных для их последующей обработки.
Чтение данных с внешнего носителя требует набор операций, связанных с *подготовкой устройства*, *операцией чтения* и собственно *пересылки данных в оперативную память*.![[Буферизация.jpg]]
Информация из файла предварительно считывается более крупными порциями в дополнительную область памяти - **буфер**, а затем из этого **буфера** переносится в оперативную память.
Размер буфера иногда существенно зависит от контекста.

Для того, чтобы избежать паузы при наполнении буфера используют **приём двойной буферизации** (при частичном освобождении первого буфера начинает заполняться второй буфер)

### Общая схема работы с файлами
Для обращения к файлу, введено понятие **дескриптора** (handler).
Дескриптор представляет собой 32-х битовое беззнаковое целое значение.
Дальнейшая работа с файлом в программе выполняется через дескриптор.
Понятие дескриптора связывают с логическим файла или файловой переменной
Связывание файловой переменной с некоторым физическим файлом (этот шаг требует обращения к устройству, где хранится файл, его активизации и считывания в буфер первой порции)

2 шаг - открытие файла (при открытии можно задавать режим доступа к данным)
3 шаг - обработка записи файла...
4 шаг - закрытие файла (при закрытии выполняется деактивация файла с целью сохранения целостности данных)
