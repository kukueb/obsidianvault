# Виртуальная машина
Любая программа, написанная на языке программирования прежде чем быть выполненной на компьютере должна быть переведена в машинный эквивалент. Будем рассматривать некий абстрактный компьютер, понимающий язык программирования и выполняющий программу. Такой компьютер называют виртуальной машиной.
### Схема выполнения программы на ВМ
#### Вход:
- Исходный код программы
- Исходные данные
#### Выход:
- Сообщения об ошибках
- Результат

Рассмотрим расширенную схему Виртуальной Машины.
#### В виртуальной машине:
Исходный код -> реальный компьютер 1 (<- Транслятор) (-> Сообщения об ошибках) -> Машинный эквивалент -> Реальный компьютер -> Сообщения об ошибках, Результат 

##### Транслятор
это программа, переводящая исходный код программы в код на машинном языке (Машинный эквивалент)

Трансляторы бывают двух видов:
1. Компиляторы
2. Интерпритаторы
Компилятор переводит весь исходный код в машинный целиком. Результат перевода сохраняет и запускает его только при отсутствии ошибок.
Интерпритатор выполняет перевод построчно, запускает результат перевода каждой строки и результат перевода не сохраняется.

#### Сообщения об ошибках (тип 1)
Синтаксические ошибки или ошибки времени трансляции (компиляции).
Такие ошибки легко формализуемы и выявляются на 99%.

#### Сообщения об ошибках (тип 2)
**Семантические ошибки** - ошибки времени выполнения. Такие ошибки практически не формализуются и должны идентифицироваться программистом.

---
Процесс трансляции из запуска машинного эквивалента может быть разделён как во времени, так и в пространстве.

## Структура простейшей программы

**Язык паскаль** является строго типизированным языком:
- Для описания переменных выделяется отдельная секция
- Переменные не могут изменять свой тип в программе

Структура на ЯП Паскаль:
```
const секция описания констант
var секция описания переменных
begin 
	операторы
end.
```

```
var a,b: integer;
begin readln():
	a:=a-b;
	writeln(a);
end.
```
Структура программы на C:
```
#директивы препроцессора
void main()
{
	
	исполняемые_операторы;
}
```

В языке C программа строится из набора функций, причём одна функция является точкой входа и называется **main**

## Директивы препроцессора (C, C++)
Управляют преобразованием текста программы до её компиляции.
**Директива** начинается с **#**
1.  ```#define``` Указывает правила замены в тексте
2. ```#include``` <имя заголовочного файла> Директива позволяет обеспечить доступ к фрагментам кода, хранящимся в библиотеках.
```
#include <math> 
#include "math.h"
``` 
$\uparrow$ одинаковые методы записи

### Числа в памяти компьютера
хранятся в прямом коде (положительные) и дополнительном (отрицательные) 
$$
5 + (-2)
$$
`0000 0101` - прямой код
`1000 0010` - прямой код
Обратный код получается из прямого инверсией, за исключением **бита знака**
`1111 1101`
Дополнительный код получается из обратного прибавлением единицы
`1111 1110` - дополнительный код
`0000 0011` - результат/прямой код

### Компиляция
Создание исполняемого файла из исходного кода состоит из 3-х этапов:
1. Препроцессор обрабатывает все директивы
2. Компилятор обрабатывает **каждый файл** с исходным кодом и создаёт объектный файл (**obj**)
3. Компоновщик соединяет все объектные файлы в единую программу.

### Структура типов языка C/C++
**Тип** - это множество значений и набор операций, определённых над этими значениями.
Типы классифицируются по критерию **скалярности** и **предопределённости**.
Тип называется **скалярным** если его значение атомарный и имеют самостоятельный смысл.
Тип называется **составным** если его значение можно разбить на отдельные скаляры.
##### Скалярные типы
1. int (Целый)
2. char (символьный)
3. wchar_t (расширенный символьный)
4. bool (логический)
5. float (вещественный)
6. double (вещественный с двойной точностью)
Тип называется **предопределённым**, если он входит в состав языка.
Тип называется **конструируемым**, если он создаётся программистом.

Для большинства типов существуют спецификаторы, уточняющие внутреннее представление и диапазон стандартных типов:
1. short (короткий)
2. long (длинный)
3. signed (знаковый)
4. unsigned (беззнаковый)

### Целочисленные типы
Положительные числа хранятся в **прямом коде**, отрицательные в **дополнительном**
Существуют **Симметричные** и **Асимметричные** типы 

| Наименование       |    Диапазон значений    | Длинна в байтах |
| ------------------ | :---------------------: | --------------- |
| int (ПК 16)        | $-2^{15} .. 2^{15} - 1$ | 2               |
| int (ПК 32)        | $-2^{31} .. 2^{31} - 1$ | 4               |
| short int          | $-2^{15} .. 2^{15} - 1$ | 2               |
| long int           | $-2^{31} .. 2^{31} - 1$ | 4               |
| unsigned short int |    $0 .. 2^{16} - 1$    | 2               |
| unsigned long int  |    $0 .. 2^{32} - 1$    | 4               |
#### Операции
1. Сложение
2. Вычитание
3. Умножение
4. Деление
5. Нахождение остатка от деления

### Описание переменных
`[Класс памяти][const] тип имя [инициализатор];`
Примеры
`int a; float x;`

**Классы Памяти**:
- **`auto`** - автоматическая локальная переменная. Иногда сложно определить тип выражения, в этом случае спецификатор **auto** предоставляет выбор типа компилятору, однако переменная с классом памяти **auto** должна быть инициализирована.
- **`extern`** - тип для глобальных переменных. То есть переменная описана в другом модуле, но присутствует в текущем.
- **`static`** - статическая переменная.
- **`register`** - работает аналогично `auto`, но память выделяется в регистрах процессора.
Каждая переменная имеет область видимости. Если в составном операторе `{}` указано описание переменной, то такой составной оператор называется **блок** и область видимости переменной начинается от строки описания до конца блока.
Если переменная описана вне функций, она называется **глобальной**. Остальные переменные по отношению к ней - **локальные**.
**Глобальная переменная** доступна в любом месте программы, если не перекрыта одноимённой **локальной переменной**.

```
int a; // глобальная переменная
void main()
{
	int b; // локальная переменная 
	extern int x; // переменная x определена в другом месте
	static int c; 
	a = 1;
	int a;
	a = 2;
	::a = 3; // Доступ к глобальной переменной
}
int x = 4;
```

### Инициализация
После определения переменной ей можно присвоить **значние** (Инициализация)
#### В C++20 имеется 3 вида инициализации:
- Нотация присваивания (assignment notaion) 
```
int age;
age = 20;
int age1 = 28;
```
- Инициализация в фигурных скобках (braced initialization)
```
int age {38};
```
- Функциональная нотация (functional notation)
```
int age (38);
int age1 {22+5};
int age2 (22+5);
int age3 = 22+5;
```
### Вещественные константы
Существуют 2 типа констант с фиксированной и плавающей точкой:
`[цифры].[цифры]`
Примеры:
`5.7, .0001, 41.`

Вид константы с плавающей точкой:
`[цифры][.][цифры]E/e[+/-][цифры]`
Примеры:
`0.5e5, .11e-5, 5E3`

такой формат называется полуэкспоненциальной формой. Сначала записывается **мантисса** (значащие цифры числа) числа, символ E - это $10^{\;некоторая\;степень}$ 

---
### Типы для представления чисел с плавающей точкой
**`float`** - вещественное число одинарной точности. В памяти занимает **4 байта** (32 бита)
В памяти компьютера 1 бит - знак, 8 бит для экспоненты и 23 бита для мантиссы.
**`double`** - вещественное число двойной точности с плавающей точкой в диапазоне $+/- 1.7E-308 \;до\; 1.7E+308$
**`long double`** - вещественное число двойной точности с плавающей точкой не менее 8 байт (64 бит). Становится больше мантисса по сравнению с `double`. 

### Символьные типы данных
**`char`** - один символ в кодировке ASCII. Занимает в памяти 1 байт.
**`wchar_t`** - расширенный символ. На Windows занимает в памяти 2 байта, на Linux 4 байта
`char8_t` - 1 байт в Unicode
`char16_t`- 2 байта в Unicode
`char32_t`- 4 байта в Unicode
