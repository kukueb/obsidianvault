### Оценивание алгоритма
любой алгоритм можно оценить по двум характеристикам:
1) Временн**а**я сложность
2) Пространственная сложность

**Временная сложность** - мера, которая учитывает время работы процессора при выполнении алгоритма.
Существует мера в *худшем случае*, то есть алгоритм оценивается при самом "плохом" наборе элементов - самое долгое время работы.
*Мера сложности в среднем* - оценивается усреднённое время работы алгоритма.
*Средневзвешенная мера* - при такой оценке операциям назначаются веса.

**Пространственная сложность** - учитывает необходимый объём памяти под используемые структуры данных в алгоритме.
 
 Временная сложность данного фрагмента
```c++
for (i = 0; i < n; i++) a[i] = ...
``` 
$t = c * n$
где $n$ - количество итераций цикла, а $c$ - некоторая константа - время затраченное на одну итерацию
Пусть время работы алгоритма выражается функцией f(n)
$f(n) = an^2 + bn + c$
Введём упрощенную функцию скорости роста $T(n)$
$T(n)=n^2$
используется $O$ нотация и время работы данного алгоритма будет оценено, как $O(n^2)$

### Метод поиска с обменом (сортировка посредством выбора)
На очередном шаге выбирается минимальный элемент, среди первых $i$ элементов и минимальный элемент меняется местами с $i$-м элементом.

### Метод пузырька
Сравнивается 2 соседних элемента и если у них нарушается упорядоченность, то они меняются местами
Кол-во сравнений - $O(n^2)$
Кол-во обменов - $O(n^2)$

### Челночная сортировка (сортировка вставками)
Идея:
	Определяется элемент, нарушающий упорядоченность и он за счёт обменов с соседними элементами перемещается в место, где восстанавливается упорядоченность.
