## Нахождение кратчайших путей между парами  вершин
Пусть дан орграф $G=(V,E)$, где к каждой дуге назначена неотрицательная стоимость (матрица стоимости $C$)
Необходимо найти кратчайшие пути для любой пары вершин. Задача решается последовательным применением алгоритма Дейкстры, переназначая источник, но есть более эффективный алгоритм - алгоритм **Флойда**

Будем рассматривать матрицу $A_{nxn}$, где n - количество вершин.
В начале $A[i,j] = C[i,j]$, причём диагональные элементы *равны нулю*.
Элементы матрицы $A$ будут содержать минимальную стоимость из $i$ в $j$

Над матрицей $A$ выполняется $n$ итераций, причём на $k$-той итерации рассматриваются только пути, проходящие через первые $k$ вершин.
$$
A_k[i,j] = min(A_{k-1}[i,j],\;A_{k-1}[i,k],\;A_{k-1}[k,j])
$$
```c++
#define n 100;
int A[n][n], C[n][n]
void Floyd() {
  int i,j,k;
  for (i = 0; i <= n; i++)
    for (j=1; j <= n; j++) A[i][j] = C[i][j];
      
  for (i=1; i <= n; i++) A[i][i] = 0;
  for (k = 1; k <= n; k++)
    for (i = 1; i <= n; i++)
      for (j = 1; j <= n; j++)
        if (A[i][k]+A[k][j] < A[i][j]) A[i][j] = A[i][k] + A[k][j];
}
```

## Нахождение собственно пути в алгоритме Флойда
Введём дополнительную матрицу $P$, в которой элемент $P[i,j]$ содержит вершину $k$, полученную при нахождении кратчайшего пути для $A[i,j]$.
Если $P[i,j] = 0$, то между $i$ и $j$ нет вершин 

```c++
#define n 100;
int A[n][n], C[n][n], P[n][n]
void Floyd() {
  int i,j,k;
  for (i = 0; i <= n; i++)
    for (j=1; j <= n; j++) {
	    A[i][j] = C[i][j];
		P[i][j] = 0;
	}
  
  for (i=1; i <= n; i++) A[i][i] = 0;
  for (k = 1; k <= n; k++)
    for (i = 1; i <= n; i++)
      for (j = 1; j <= n; j++)
        if (A[i][k]+A[k][j] < A[i][j]) {
	        A[i][j] = A[i][k] + A[k][j];
	        P[i][j] = k;
		}
}
```

## Вывод кратчайшего пути

```c++

  int k = P[i][j];
```

## Поиск центра ориентированного графа
Пусть $v$ - произвольная вершина орграфа $G=(V,E)$.
*Эксцентриситет* вершины $v$ определяется как 
	max {наименьшая длина пути от вершины $w$ до вершины $v$}

Для всех вершин необходимо вычислить наименьшие пути в некоторую вершину (из $a$ в $b$  - 1, $c$ в $b$  - 3, $d$ в $b$  - 1, $e$ в $b$  - 6)
Из наименьших длин выбрать максимальную (6), это *эксцентриситет* для вершины $b$

Центром орграфа $G$ называется вершина с наименьшим эксцентриситетом, то есть вершина для которой максимальное расстояние до других вершин минимально.

### Алгоритм нахождение центра графа
Пусть $C$ - матрица стоимостей орграфа $G$
1. Применить алгоритм Флойда к матрице $C$ для вычисления матрицы кратчайших путей $A$.
2. Найти максимальное значение в каждом столбце $i$ матрицы $A$. Это значение есть эксцентриситет вершины $i$.
3. Найти вершину с наименьшим эксцентриситетом. Это и есть центр орграфа G.

## Обход ориентированных графов
В задачах на графах необходим способ систематического обхода вершин.

Существует 2 способа обхода:
- поиск в глубину
- поиск в ширину

### Поиск в глубину
Для представления вершин, смежных с вершиной $v$, можно использовать список смежности $L[V]$, а для определения вершин, которые ранее посещались - массив `mark` (метка), чьи элементы будут принимать только два значения: `visited` и `unvisited`.

```c++
for (v=1; v<= n; v++) 
  mark[v] = unvisited;
  for (v=1; v<=n; v++)
    if (mark[v] == unvisited) dfs(v);

// Процедура поиска в глубинус
void dfs(вершина v) {
  вершина w;
  mark[v] = visited;
  for каждая вершина w из списка L[v] // for (j = 1; j < n; j++)
    if (mark[w] == unvisited) // if (mark[j] == 0 && A[v][j] != 1000)
      dfs(w); {строка *} // dfs(j);
}
```

## Сильная связность
Пусть $G=(V,E)$ - орграф. Множество вершин $V$ разбивается на классы эквивалентности $V_j$, $1 <= i <= r$, так чтобы вершины $v$ и $w$ будут эквивалентны тогда и только тогда, когда существуют пути из вершины $v$ в вершину $w$ и из вершины $w$ в вершину $v$. Пусть $E_i$, $1 <= i <= r$, - множество дуг, которые начинаются и заканчиваются в множестве вершин $V_i$. Тогда графы $G_i=(V_i,E_i)$ называются *сильно связными компонентами* орграфа $G$. Ориентированный граф, состоящий из одной сильно связной компоненты, называется *сильно связным*

### Алгоритм нахождения сильно связных компонент
1. Выполнить поиск в глубину на графе $G$. Вершинам присвоить номера в порядке обхода процедуры `dfs`, т.е. присвоение номеров вершинам происходит после `{строка *}`
2. Создать новый ориентированный граф $G_r$, путём обращения направления всех дуг графа $G$. (Транспонировать матрицу стоимостей)
3. Выполнить поиск в глубину на графе $G_r$, начиная с вершины с наибольшим номером, присвоенном на шаге 1. Если проведённый поиск не охватывает всех вершин, то начать новый поиск с вершины, имеющей наибольший номер среди оставшихся вершин.
4. Каждое дерево в полученном основном лесу является сильно связной компонентой орграфа $G$.
