### Функция compare()
функция compare возвращает 0, если две строки равны. Если текущая строка больше, то возвращается число больше 0. Если текущая строка меньше, то возвращается число меньше 0 (-1)

```c++
string tom {"Tom"};
string person {"Tom"};

int result = tom.compare(person);
cout << result << endl; // 0 - строки равны
```

```c++
string tom {"Tom"};
string bob {"Bob"};
string sam {"Sam"};

int result = sam.compare(bob); // 1 - первая строка (sam) больше второй (bob)
cout << result << endl;
result = sam.compare(tom); // -1 - первая строка (sam) меньше второй (tom)
cout << result << endl;
```

## Ориентированные графы
**Ориентированный граф** (**орграф**)  $G = (V, E)$ состоит из множества вершин $V$ и множества дуг $Е$
Дуга, ведущая от вершины $v$ к вершине $w$ ($v \to w$). Вершина $w$ называется **смежной** с вершиной $v$
Путём в графе называется последовательность $v_{1}, \dots, v_{n}$ такая что существуют дуги $v_{1} \to v_{2}$ $v_{2} \to v_{3}$ $,\; \dots\; ,$ $v_{n-1} \to v_{n}$
Если существует дуга $v_{1}\to v_{n}$ то путь называется **циклическим**

Граф называется нагруженным, если у дуги и/или вершины есть **вес**, то есть некоторая дополнительная информация
### Представление ориентированных графов
Предположим, что множество вершин $V=\{1,2,\dots,n\}.$ 
Граф представляется с помощью **матрицы смежности**, то есть матрицы $A_{n X n}$ со значением где $A[i, j] = true$ тогда и только тогда, когда существует дуга из вершины $i$ в $j$. (false, если вершины нет)

Если граф нагружен, то при наличии дуги из $i$ в $j$ в элементе $A[i,j]$ указывается вес дуги.
В этом случае матрица смежности часто называется **матрицей стоимости**

|       | 1   | 2   | 3   | 4   |
| ----- | --- | --- | --- | --- |
| **1** |     | a   |     | b   |
| **2** | a   |     | b   |     |
| **3** |     | b   |     | a   |
| **4** | b   |     | a   |     |
Недостатком матричного представления является использование большого объёма памяти *(если объём $V$ большой | много вершин*, а дуг мало)

#### Список смежности
При малом количестве дуг удобно рассматривать массив заголовков, где каждый элемент является указателем на **список смежности** соответствующей вершины.

Для просмотра множества смежных вершин необходимы 3 абстрактных оператора
- FIRST(v) возвращает индекс первой вершины, смежной с вершиной v. Если вершина v не имеет смежных вершин, то возвращается "нулевая" вершина $\Lambda$.
- NEXT(v, i) возвращает индекс вершины, смежной с вершиной v, следующий за индексом i. Если i - это индекс последней вершины, смежной с вершиной v, то возвращается $\Lambda$
- VERTEX(v, i) возвращает вершину с индексом i из множества вершин, смежных с v.

```c++
int FIRST(int v) {
	for (int i = 1; i <= n; i++)
		if (A[v][i]) return i;
	cout << "вершина v не имеет смежных вершин";
	return 0;
}

int NEXT(int v, int i) {
	for (int j = i+1; j <= n; j++)
		if (A[v][j]) return j;
		cout << "вершина v не имеет смежных вершин";
	return 0;
}
```

```c++
i = FIRST(v);
while (i) {
	w = VERTEX(v, i);
	// действия с вершиной w
	i = NEXT(v,i);
}
```

### Задача нахождения кратчайшего пути
Пусть есть ориентированный граф $G=(V,E)$ у которого все дуги имеют неотрицательные веса. Одну из вершин определим как источник. Задача состоит в нахождении стоимости кратчайших путей от источника ко всем другим вершинам графа.

Задача решается алгоритмом Дейкстры (Edsger W. Dijkstra). Алгоритм строит множество вершин $S$ для которых пути от источника уже известны. На каждом шаге к множеству $S$ добавляется та из оставшихся вершин, расстояние до которой от источника меньше чем до других вершин. Путь, пролегающий через эту вершину, называется **особым**, а его длина хранится в массиве $D$. 
Пусть $V = \{1,2, \dots, n\}$, вершина $1$ является источником.
Матрица стоимости содержит веса дуг. Если дуги нет между вершиной $i,\; j$ , то $A[i][j]$ равен достаточно большому числу

```c++
S = {1};
for (i=2; i<= n; i++)
	D[i] = A[1][i]; {инициализация D}
for (i = 1; i <= n-1; i++)
{
	Выбор из множества V\S такой вершины w, что значение D[w] минимально;
		добавить w к множеству S;
		for каждая строка v из множества V\S
			D[v] = min(D[v], D[w] + A[w][v])
}
end;
```

```c++
#define SIZE 6
int main() {
	int a[SIZE+1][SIZE+1]; //матрица связей
	int d[SIZE+1]; // минимальное расстояние
	int v[SIZE+1]; // посещённые вершины
	int w, min;
	// ... ввод матрицы связей, если нет ребра a[i][j] = 1000
	// инициализация вершин и расстояний
	
	for (int i = 2; i <= SIZE; i++) {
		d[i] = a[1][i];
		v[i] = 1;
	}
	d[1] = 0

	// Шаг алгоритма
	do {
		w = 1000;
		min = 1000;
		for (int i = 2; i <= SIZE; i++) {
			// Если вершину еще не обошли и вес меньше min	
			if ((v[i] == 1) && (d[i] < min)) {
				// Переприсваиваем значения
				min = d[i];
				w = i; // вершина w
			}
		}
		if (w != 1000) {
			for (int i = 2; i <= SIZE; i++) {
				if (min + a[w][i] < d[i]) d[i] = min + a[w][i];
			}
			v[w] = 0; // отменили w как посещённую
		}
	} while (w < 1000);
}
cout << "Кратчайшие расстояния до вершин: \n";
for (int i = 1; i <= SIZE; i++)
	cout << "от 1й вершины к " << i << "расстояние = " << d[i] << endl;
```
